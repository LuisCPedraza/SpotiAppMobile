Taller: Creación de Spotiapp Mobile con Flutter  Objetivo: Desarrollar una aplicación móvil que consuma la API de Spotify para mostrar nuevos lanzamientos y buscar artistas, implementando servicios, modelos de datos y manejo de estados. Fase 1: Configuración del Entorno de Spotify (Backend)Antes de programar, es imperativo configurar el acceso a los datos: • Registro de App: Acceder al Spotify for Developers Dashboard y crear una nueva aplicación llamada "Spotiapp Mobile".  • Credenciales: Obtener el Client ID y el Client Secret.  • Configuración de Redirect URI: Para móviles, configurar http://localhost:8888/callback (o el esquema personalizado de la app). Fase 2: Arquitectura del Proyecto Mobile A diferencia de Angular, en Flutter organizaremos los componentes de la siguiente manera: • Servicios: spotify_service.dart para manejar peticiones HTTP.  • Pantallas (Views): home_page.dart, search_page.dart y artist_page.dart.  • Widgets Globales: Componentes reutilizables como tarjetas de artistas y cargadores (loading).  Fase 3: Desarrollo de Servicios (Peticiones HTTP) 1. Autenticación: Implementar una función para obtener el Access Token mediante el ﬂujo de Client Credentials. 2. Consumo de API: • Nuevos Lanzamientos: Crear el método getNewReleases() apuntando a v1/browse/new-releases • Búsqueda: Crear el método getArtistas(String termino) usando el endpoint v1/search?q=...&type=artist.  Fase 4: Interfaz de Usuario (UI) 1. Home Page: o Implementar un GridView para mostrar los nuevos lanzamientos. o Mostrar la imagen del álbum, el nombre y el artista. 2. Search Page: o Incluir un TextField que dispare la búsqueda mientras el usuario escribe o al presionar "Enter". o Usar un Loading Widget (como un CircularProgressIndicator) mientras se procesa la petición. 3. Pipes Adaptados (Null Safety): Crear una función que valide si la imagen del artista es nula; si es así, mostrar una imagen por defecto (assets/img/noimage.png).  Fase 5: Detalle del Artista y Reproducción 1. Navegación: Al tocar una tarjeta, navegar a ArtistPage pasando el id del artista. 2. Top Tracks: Consultar y listar las 10 canciones más populares en un ListView. 3. Audio Preview: Implementar un botón de reproducción que consuma la URL de preview_url (Spotify permite 30 segundos de vista previa).
